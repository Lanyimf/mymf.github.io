# 場址篩選邏輯（褐地開發應用版，基礎篩選）

> 這份筆記用 Dataview 做場址的**基礎篩選**，純粹用 Obsidian 筆記的 frontmatter 計算，不會改動網頁。
>
> **重要前提**：本網站主軸是**褐地開發應用**——資料庫裡的 342 筆場址全部都是目前列管中的污染場址（無一筆已解除列管）。因此「是否列管」不是用來把場址判定「不通過」的條件，而是用來標註「此用途需等整治完成解除列管後才能正式申請」。
>
> 篩選規則：
> 1. 若場址沒有 `type_code`（使用地類別代碼）→ **資料不足，無法判定**（結果清單中隱藏，僅計入統計）
> 2. 若場址 `type_code` 與規則 `type_code` 不一致 → **不通過**（使用地類別不符，跟列管狀態無關，整治完成也不會通過；結果清單中隱藏，僅計入統計）
> 3. 使用地類別符合，但規則硬性要求非列管場址，而此場址目前仍列管中 → **條件式通過：需待整治完成解除列管後才可申請**（會顯示）
> 4. 使用地類別符合，規則本身不要求非列管場址 → **通過基礎篩選**（僅基礎篩選，非完整評估；會顯示）
>
> 結果清單**只列出第3、4種（通過/條件式通過），不顯示「不通過」與「資料不足」**，但會在上方統計一行說明被隱藏的筆數。每個顯示的項目都會列出「已自動驗證的條件」與「尚待人工查核的條件」兩段。
>
> **財務分數（僅供相對排序參考，不是真實財務預測）**：分數 = 公告現值（元/m²）× 場址面積（m²），條件式通過再乘以 0.8 的風險折減。這只是用我們手上唯二的地價資料做的簡化排序指標，**沒有納入營收、租金、營造成本、市場需求**等任何真實財務建模要素，請勿當作投資評估依據。目前資料庫裡只有少數場址查到公告現值（其餘場址會顯示「缺財務資料」），需要再花時間逐筆查詢補齊。

## 使用方式

修改下面程式碼最上面的兩個變數：
- `selectedRuleCode`：填一個規則代碼（如 `"EE-12"`），會列出所有符合的場址
- `selectedLandId`：填一個場址編號（如 `"B10043"`），會列出該場址符合哪些規則

兩個都可以留空字串 `""` 不查，或都填會分別各自顯示一段結果。

```dataviewjs
// ====== 在這裡修改查詢條件 ======
const selectedRuleCode = "EE-12"   // 例如 "EE-12"，留空 "" 表示不查
const selectedLandId = ""          // 例如 "B10043"，留空 "" 表示不查
// ================================

const POLLUTED_STATUSES = ["公告為控制場址", "公告為整治場址"]

function isPollutedSite(land) {
  return POLLUTED_STATUSES.includes(land.status)
}

// 場址清單
const lands = dv.pages('"lands"').map(p => ({
  id: p.id,
  name: p.name,
  status: p.status,
  type_code: p.type_code,
  area_m2: p.area_m2,
  announced_current_value: p.announced_current_value, // 例如 "175000 元/平方公尺"
  file: p.file,
}))

// 規則清單：requires_clean_site 透過讀取規則內文文字判斷；同時抓硬性門檻/加權指標條數，與條件條列文字
const ruleFiles = dv.pages('"rules"')
const rules = []
for (const r of ruleFiles) {
  const text = await dv.io.load(r.file.path)
  // 抓出「法規資格條件」表格裡的條目文字（# | 指標項目 | 類型 | ...）
  const criteriaRows = [...text.matchAll(/^\|\s*\d+\s*\|\s*([^|]+?)\s*\|\s*(硬性門檻|加權指標)\s*\|/gm)]
    .map(m => ({ name: m[1].trim(), type: m[2].trim() }))
  rules.push({
    eval_code: r.eval_code,
    type_code: r.type_code,
    type_name: r.type_name,
    use_item: r.use_item,
    requires_clean_site: text.includes("污染控制場址或整治場址"),
    criteriaRows,
    file: r.file,
  })
}

// 解析「175000 元/平方公尺」這種字串取出數字
function parseValue(str) {
  if (!str) return null
  const m = String(str).match(/[\d.]+/)
  return m ? parseFloat(m[0]) : null
}

function evaluate(land, rule) {
  // 使用地類別是真正的篩選條件，先比對
  if (!land.type_code) {
    return { result: "資料不足", reason: "場址缺少使用地類別代碼（type_code）" }
  }
  if (land.type_code !== rule.type_code) {
    return { result: "不通過", reason: `場址使用地類別為「${land.type_code}」，與規則要求「${rule.type_code}」不符` }
  }

  // 已自動驗證的條件 vs 尚待人工查核的條件（取自規則本身的硬性門檻/加權指標清單）
  const verified = [`使用地類別相符（${rule.type_code}）`]
  const pending = []
  let result, reason

  if (rule.requires_clean_site && isPollutedSite(land)) {
    verified.push(`非列管狀態檢查：尚未通過（目前為「${land.status}」，需整治完成解除列管）`)
    result = "條件式通過"
    reason = `使用地類別符合（${rule.type_code}），但本用途硬性要求非列管場址；此場址目前列管狀態為「${land.status}」，需待整治完成解除列管後才可正式申請`
  } else {
    if (rule.requires_clean_site) verified.push("非列管狀態檢查：已符合（非列管場址要求）")
    result = "通過基礎篩選"
    reason = `使用地類別符合（${rule.type_code}），且本用途不要求非列管場址`
  }

  // 其餘規則條件（區位、空間、其他硬性門檻等）目前沒有對應資料可自動驗證
  for (const c of rule.criteriaRows) {
    if (c.name.includes("使用地類別") || c.name.includes("使用分區")) continue // 已併入上面的使用地類別檢查
    if (c.name.includes("污染場址身份") || c.name.includes("污染")) continue // 已併入上面的列管檢查
    pending.push(`${c.name}（${c.type}）：尚無對應資料，需人工查核`)
  }

  // 財務分數（僅為簡化相對排序用，非真實財務預測）
  const unitValue = parseValue(land.announced_current_value)
  let financeScore = null
  let financeNote = ""
  if (unitValue != null && land.area_m2) {
    financeScore = unitValue * land.area_m2
    if (result === "條件式通過") financeScore *= 0.8 // 尚需整治解除列管，給予風險折減，純粹經驗法則
    financeNote = `公告現值 ${unitValue} 元/m² × 面積 ${land.area_m2} m²${result === "條件式通過" ? " ×0.8（列管中風險折減）" : ""} ≈ ${Math.round(financeScore).toLocaleString()} 元`
  } else {
    financeNote = "缺少公告現值資料，無法計算"
  }

  return { result, reason, verified, pending, financeScore, financeNote }
}

// 把結果排序：有財務分數的依分數高到低；沒有分數的排在最後（仍保留通過/條件式通過的順序）
function sortByScore(items) {
  return items.sort((a, b) => {
    if (a.financeScore != null && b.financeScore != null) return b.financeScore - a.financeScore
    if (a.financeScore != null) return -1
    if (b.financeScore != null) return 1
    const order = { "通過基礎篩選": 0, "條件式通過": 1 }
    return order[a.result] - order[b.result]
  })
}

// ---- 模式一：依規則查場址（隱藏不通過／資料不足，僅列可行場址，依財務分數排序）----
if (selectedRuleCode) {
  const rule = rules.find(r => r.eval_code === selectedRuleCode)
  if (!rule) {
    dv.paragraph(`❌ 找不到規則代碼「${selectedRuleCode}」`)
  } else {
    dv.header(3, `規則 ${rule.eval_code}（${rule.type_name}－${rule.use_item}）可行的場址（依財務分數排序）`)
    let items = lands.map(land => ({ land, ev: evaluate(land, rule) }))
    const hiddenFail = items.filter(i => i.ev.result === "不通過").length
    const hiddenNoData = items.filter(i => i.ev.result === "資料不足").length
    items = items.filter(i => i.ev.result === "通過基礎篩選" || i.ev.result === "條件式通過")
    items = sortByScore(items.map(i => ({ ...i.ev, land: i.land })))

    dv.paragraph(`共 ${items.length} 筆場址符合此用途（另有 ${hiddenFail} 筆使用地類別不符、${hiddenNoData} 筆缺少使用地類別資料，已隱藏不顯示）。`)
    for (const it of items) {
      dv.header(4, `${it.land.file.link}　${it.result === "條件式通過" ? "🟡 條件式通過" : "🟢 通過基礎篩選"}　${it.financeScore != null ? `財務分數約 ${Math.round(it.financeScore).toLocaleString()} 元` : "（缺財務資料）"}`)
      dv.paragraph(`**通過原因**：${it.reason}`)
      dv.paragraph(`**財務分數試算**：${it.financeNote}`)
      if (it.verified.length) dv.list(it.verified.map(v => `✅ ${v}`))
      if (it.pending.length) dv.list(it.pending.map(v => `⏳ ${v}`))
    }
  }
}

// ---- 模式二：依場址查規則（隱藏不通過／資料不足，僅列可行用途，依財務分數排序）----
if (selectedLandId) {
  const land = lands.find(l => l.id === selectedLandId)
  if (!land) {
    dv.paragraph(`❌ 找不到場址編號「${selectedLandId}」`)
  } else {
    dv.header(3, `場址 ${land.id}（${land.name}）可行的用途（依財務分數排序）`)
    let items = rules.map(rule => ({ rule, ev: evaluate(land, rule) }))
    const hiddenFail = items.filter(i => i.ev.result === "不通過").length
    const hiddenNoData = items.filter(i => i.ev.result === "資料不足").length
    items = items.filter(i => i.ev.result === "通過基礎篩選" || i.ev.result === "條件式通過")
    items = sortByScore(items.map(i => ({ ...i.ev, rule: i.rule })))

    dv.paragraph(`共 ${items.length} 條用途符合此場址（另有 ${hiddenFail} 條使用地類別不符、${hiddenNoData} 條因場址缺少使用地類別資料無法判定，已隱藏不顯示）。`)
    for (const it of items) {
      dv.header(4, `${it.rule.file.link}　${it.result === "條件式通過" ? "🟡 條件式通過" : "🟢 通過基礎篩選"}　${it.financeScore != null ? `財務分數約 ${Math.round(it.financeScore).toLocaleString()} 元` : "（缺財務資料）"}`)
      dv.paragraph(`**通過原因**：${it.reason}`)
      dv.paragraph(`**財務分數試算**：${it.financeNote}`)
      dv.paragraph(`**已自動驗證的條件**：`)
      if (it.verified.length) dv.list(it.verified.map(v => `✅ ${v}`))
      dv.paragraph(`**尚待人工查核的條件**（規則內還有區位、空間等條件未自動驗證）：`)
      if (it.pending.length) {
        dv.list(it.pending.map(v => `⏳ ${v}`))
      } else {
        dv.paragraph("（此規則沒有其他額外條件需查核）")
      }
    }
  }
}

if (!selectedRuleCode && !selectedLandId) {
  dv.paragraph("請在程式碼最上面填入 `selectedRuleCode` 或 `selectedLandId` 來查詢。")
}
```

## 統計摘要

```dataviewjs
const POLLUTED_STATUSES = ["公告為控制場址", "公告為整治場址"]
const lands = dv.pages('"lands"')
const withTypeCode = lands.where(l => l.type_code).length
const polluted = lands.where(l => POLLUTED_STATUSES.includes(l.status)).length

dv.paragraph(`場址總數：${lands.length} 筆`)
dv.paragraph(`有使用地類別代碼（type_code）：${withTypeCode} 筆（${lands.length - withTypeCode} 筆缺少，基礎篩選對這些場址只能回傳「資料不足」）`)
dv.paragraph(`列管為污染控制/整治場址：${polluted} 筆（本網站主軸為褐地開發應用，這 ${polluted} 筆都是目前仍在列管中的場址；對於硬性要求非列管場址的用途，篩選結果會標示「條件式通過：需待整治完成解除列管後才可申請」，而不是直接判定不通過）`)
```
